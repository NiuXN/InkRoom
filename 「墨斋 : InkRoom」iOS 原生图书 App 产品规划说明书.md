# 📖 「墨斋 / InkRoom」iOS 原生图书 App 产品规划说明书

## 一、 产品概述
*   **产品定位：** 一款基于 **SwiftUI + SwiftData** 纯原生开发的、专注于本地书源与个人云端资产的“高质感私人书房”阅读器。
*   **核心特色：** 智能封面美化、禅意笔记输出、以及**完全离线、无广告的 iOS 原生高质量听书**。
*   **技术特色：** 100% 纯原生、零后台服务器成本（利用 iCloud）、极度省电、零数据隐私收集，遵循“小而美”与“小而获利”的独立开发原则。

---

## 二、 核心功能清单与模块规划

### 1. 📚 模块一：智能书房 (My Library) —— 资产管理
> **设计目标：** 改变传统阅读器“导入即变乱”的痛点，给用户实体书架般的视觉享受。

*   **1.1 智能元数据刮削 (核心亮点，第一阶段仅做基础解析)：**
    *   用户导入 `.epub` 或 `.txt` 时，App 自动解析内部的 ISBN、书名或作者名。
    *   后台自动请求公开图书 API，自动补全**高清图书封面、作者简介、出版年份及官方摘要**。
    *   *MVP 阶段目标：先保证书名、作者等基础信息可读；完整刮削与多源 fallback 在第二阶段完善。*
*   **1.2 多维书架管理：**
    *   **视图切换：** 支持“精美海报墙（Grid）”与“高效列表（List）”双视图。
    *   **智能分组：** 系统自动归类为：最近阅读、未读、在读、已读。
    *   **自定义标签 (第一阶段不做)：** 用户可自由创建分类标签（如：产品经理、科幻、历史）。
*   **1.3 多渠道便捷传书：**
    *   **Wi-Fi 局域网传书：** 开启后在同 Wi-Fi 的电脑浏览器输入本地 IP，即可批量拖拽上传。
    *   **iOS 系统集成：** 完美支持 AirDrop（隔空投送）、系统“文件 (Files)”应用直接调用墨斋打开。

### 2. 📖 模块二：沉浸阅读引擎 (Reader Engine) —— 极致输入
> **设计目标：** 像素级的排版微调，提供最纯净、无任何广告与社交打扰的沉浸环境。

*   **2.1 原生排版美学：**
    *   内置 4 款精心挑选的中文字体（如：官派宋体、人文楷体、现代黑体等）(第一阶段先用系统默认字体，第二阶段引入)。
    *   提供字号、行距、字距、段落间距的黄金比例滑块，支持动态字体（Dynamic Type）。
*   **2.2 双模式翻页：**
    *   **经典模式 (第一阶段不做)：** 基于 Metal 渲染的拟真仿真翻页动画（复刻实体书质感）。
    *   **现代模式：** 类似网页的无缝垂直滚动（适合快速扫读）。
*   **2.3 硬核 TXT 智能断章：**
    *   针对大体积 TXT 纯文本小说，在本地运行高效正则表达式，**毫秒级自动解析并提取章节目录**（如：“第一章”、“第 X 节”），彻底解决中文 TXT 乱码和无法跳转的问题。

### 3. 🎧 模块三：离线禅意听书 (Audio Engine) —— 降维打击
> **设计目标：** 针对巨头平台（番茄/七猫）听书全是广告、耗流量的死穴，利用 iOS 原生能力打造纯净听书。

*   **3.1 iOS 原生自然人声 (TTS，第一阶段不做)：**
    *   调用 iOS `AVSpeechSynthesizer` 引擎，直接适配苹果官方最新内置的**高质量、自然语义人声**（如中文的高级人声：嘉婧、博文等）。
    *   **全本地运行：** 声音拟真度极高，且不消耗任何网络流量。
    *   *MVP 阶段聚焦阅读本身，TTS 听书整体放入第二阶段。*
*   **3.2 语境双向同步 (TTS Sync，第一阶段不做)：**
    *   听书时，阅读界面自动随语音进度滚动，并以柔和色块高亮当前朗读的句子。
    *   **点选朗读：** 在阅读界面用手指双击或长按任意句子，听书进度立即跳转至该句开始播放。
*   **3.3 锁屏常驻与媒体控制 (第一阶段不做)：**
    *   完美接入 iOS 系统 `Now Playing` 控件，支持锁屏界面、控制中心、Apple Watch、蓝牙耳机的切歌键进行播放/暂停/快进。
    *   后台常驻播放，切换到其他 App 或锁屏时绝不断播。
*   **3.4 催眠白噪音与定时 (第一阶段不做)：**
    *   **催眠定时器：** 支持“定时 15/30/60 分钟”或“播放完本章自动停止”，停止时声音缓缓淡出（Fade-out）。
    *   **氛围伴奏：** 允许在听书人声下混入极低音量的本地白噪音（如：雨落窗台、炉火劈啪、京都寺庙深钟）。

### 4. ✍️ 模块四：知识管理与输出 (Notes & Export) —— 优雅输出
> **设计目标：** 让阅读在应用内形成完美的闭环，满足深度读者的整理与分享欲。

*   **4.1 禅意金句卡片 (第一阶段不做)：**
    *   划线或添加读书想法（Thoughts）时，可一键生成设计感极佳的**分享长图/卡片**。
    *   提供 5-6 款极简的排版模版（支持毛玻璃背景、东方传统色渐变、全木纹背景等），方便一键保存或分享。
*   **4.2 生产力友好导出 (第一阶段不做)：**
    *   支持一键将整本书的划线、笔记汇总导出为标准的 **Markdown (.md)** 格式。
    *   导出格式完美兼容 **Notion、Obsidian、Logseq** 等现代主流知识库软件。

### 5. ☁️ 模块五：iCloud 安全同步 (Sync & Privacy) —— 护城河
*   **5.1 零服务器架构 (第一阶段不做)：**
    *   完全基于苹果官方 **CloudKit (iCloud)** 存储，用户的数据直接保存在自己的苹果云端。
    *   **多端同步：** 自动在用户的 iPhone、iPad、Mac 之间实时同步阅读进度、听书锚点以及书架列表。
    *   *MVP 阶段先保证本地单机体验稳定；iCloud 同步放入第三阶段。*
*   **5.2 极致隐私承诺：**
    *   100% 零数据收集。不需要注册账号，不读取设备 IDFA（广告标识符），应用体积保持在 20MB 以内，给用户最大的安全感。

## 三、 竞品功能对比与个人开发优势

### 3.1 核心竞品对比

以下从阅读本质体验、隐私安全、商业模式三个关键维度，将「墨斋」与市面主流阅读 App 进行对比：

| 对比维度 | 微信读书 | 番茄小说 / 七猫 | Apple Books | **墨斋 InkRoom** |
| :------- | :------- | :-------------- | :---------- | :---------------- |
| **核心定位** | 社交化在线阅读 | 免费网文 + 广告变现 | 苹果官方书城 | 本地书源私人书房 |
| **广告体验** | 无开屏广告，但有活动/推荐推送 | 听书、翻页、任务环节均有广告 | 无广告 | **零广告，零推送** |
| **离线能力** | 需登录，部分书籍可下载 | 需联网看广告才能解锁章节 | 支持本地导入 | **纯本地运行，断网可读可听** |
| **本地书源** | 不支持本地 `.epub` / `.txt` 导入 | 不支持本地书籍 | 支持 `.epub` | **支持 `.epub` / `.txt` 全格式** |
| **听书体验** | 在线语音，受网络影响 | 广告插播，音质一般 | 不支持中文 TTS 听书 | **原生高质量人声，无广告插播** |
| **数据隐私** | 需微信账号，阅读数据上传 | 需手机号，行为数据用于广告推荐 | 绑定 Apple ID | **不收集任何用户数据，无需注册** |
| **跨设备同步** | 依赖微信生态 | 依赖账号体系 | iCloud 同步 | **iCloud 原生同步，无服务器成本** |
| **商业模式** | 会员订阅 + 版权分销 | 广告 + 付费解锁 | 图书购买 | **买断制 / 订阅制，一次性获得全部能力** |

### 3.2 个人独立开发的差异化优势

相比大厂产品，「墨斋」作为个人独立开发作品，核心优势在于**不做平台、只做工具**，把阅读体验交还给用户：

*   **零广告、零打扰：** 没有开屏广告、没有激励视频、没有算法推荐流，打开即读，听书不被打断。
*   **数据完全私有：** 不收集手机号、不读取设备 ID、不上传阅读记录，所有数据仅存储在用户自己的 iCloud 中。
*   **本地资产优先：** 支持用户已有的 `.epub`、`.txt` 藏书，而不是把用户锁在平台内容库里。
*   **买断制更诚实：** 一次付费即可解锁全部能力，没有章节解锁、没有代币充值、没有抽卡式付费。
*   **纯原生更省电：** 基于 SwiftUI + SwiftData + AVSpeechSynthesizer 构建，不依赖后台服务器，不常驻网络连接，显著延长续航。
*   **专注阅读本质：** 砍掉社交、直播、社区、运营活动等无关功能，只保留读书、听书、笔记三条主线。
*   **快速响应迭代：** 独立开发决策链路短，用户反馈可直接转化为产品改进，无需跨部门排期。

## 四、 产品定位、竞争壁垒与风险分析

### 4.1 一句话定位

> **墨斋 = 没有广告、不收集数据、支持本地藏书买断制的 iOS 原生阅读工具。**

不做内容平台，不做社交社区，只解决“用户已有书籍怎么读得舒服”的问题。

### 4.2 核心竞争壁垒

| 壁垒类型 | 具体表现 |
| :------- | :------- |
| **商业模式壁垒** | 买断制 + 无广告，与大厂“免费 + 广告/内容付费”形成直接反差 |
| **隐私信任壁垒** | 零数据收集、无需注册、iCloud 私有存储，建立用户长期信任 |
| **技术成本壁垒** | 纯原生 + 零服务器，长期运维成本极低，可持续小规模运营 |
| **场景粘性壁垒** | 离线 TTS 听书 + 本地书源，覆盖通勤、睡前、飞行等无网络场景 |
| **品牌调性壁垒** | “小而美”独立应用气质，吸引厌倦大厂繁杂功能的 iOS 核心用户 |

### 4.3 主要风险与应对

| 风险点 | 影响程度 | 应对策略 |
| :----- | :------- | :------- |
| **目标用户规模偏小** | 中 | 聚焦 iOS 硬核读者，不追求大众市场；通过买断制提升单用户价值 |
| **元数据刮削依赖外部 API** | 高 | MVP 阶段先保证基础解析；第二阶段引入多源 fallback + 本地缓存 |
| **EPUB/TXT 排版与断章技术门槛** | 高 | 优先做垂直滚动模式，放弃第一阶段仿真翻页，降低初期复杂度 |
| **Freemium 卡点影响口碑** | 中 | 免费版限制“同时在读 3 本 + 每日听书 30 分钟”，但阅读本身不强制付费 |
| **大厂跟进纯净阅读赛道** | 低 | 独立开发者决策更快、更专注，可通过社区反馈快速迭代建立先发优势 |

### 4.4 第一阶段明确不做的功能清单

为保证 MVP 在 1-2 个月内可交付，以下功能放入第二、三阶段：

*   **听书相关：** TTS 人声朗读、语境双向同步、锁屏 Now Playing 控制、白噪音伴奏、定时关闭
*   **视觉高级：** Metal 仿真翻页动画、4 款定制中文字体、复杂金句卡片模板
*   **输出与同步：** Markdown 笔记导出、金句卡片生成、iCloud 多端同步
*   **书架高级：** 自定义标签、完整多源元数据刮削

MVP 阶段仅聚焦：**本地导入 → 基础书架 → 稳定文本渲染与垂直滚动阅读 → 阅读进度记忆**。

## 五、 商业化设计（Freemium 模式规划）

作为个人独立开发，为了保证纯净的品牌形象，**坚决不引入任何第三方广告 SDK**。采用“免费版建立口碑，Pro 版卡点盈利”的模式：

| 功能模块     | 👥 免费版 (基础体验)          | 💎 Pro 高级版 (解锁极致)                      |
| :----------- | :--------------------------- | :------------------------------------------- |
| **书架容量** | 同时在读数量限制为 **3 本**  | **无限制**，支持无限建架和囤书               |
| **排版系统** | 基础字号、系统默认字体       | 解锁**精选高级字体**、精美主题皮肤           |
| **听书功能** | **每天限制听书 30 分钟**     | **无限制听书**，解锁白噪音伴奏、锁屏快捷控制 |
| **笔记输出** | 基础复制、带有应用水印的卡片 | **一键 Markdown 导出**、高画质无水印卡片     |
| **云端同步** | 仅限本地单机存储             | **全自动 iCloud 跨设备多端无缝同步**         |

### 定价方案建议：
*   **月度订阅：** ¥ 6 / 月（适合短期尝试）
*   **年度订阅：** ¥ 38 / 年（主力定价，极具性价比）
*   **终身买断：** ¥ 68 或 ¥ 98（独立应用用户最爱的付费形式，能快速在上架初期回收开发成本）

## 六、 独立开发实施路径（MVP 阶段划分）

*   **第一阶段（第 1-2 个月）：MVP 最小可行性产品**
    *   搞定基于 SwiftUI 的文本渲染与平滑滚动。
    *   实现本地 `.epub` 和 `.txt` 的基础解析与本地 Wi-Fi 传书。
    *   做出最核心的、干净的书架。
*   **第二阶段（第 3 个月）：听书与高级卡片**
    *   集成 `AVSpeechSynthesizer`，实现基础的后台听书与定时关闭。
    *   开发划线和金句卡片生成器。
*   **第三阶段（第 4 个月）：iCloud 同步与商业化**
    *   接入 SwiftData + CloudKit 实现多设备同步。
    *   接入 Apple In-App Purchase（内购），配置订阅制和买断制，准备上架。
